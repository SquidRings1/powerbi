📋 Dashboard Structure

3 Pages

📊 Page 1: Vue Globale (Global Overview)

  • 4 KPI Cards (Total Apps, Critiques, Sur-Conso, OK)
  • Donut Chart (Status distribution: 4 categories)
  • Line Chart (14-month evolution trend)
  • Top Problems Table (8 worst apps)

🔍 Page 2: Détail App (Application Drill-Down)

  • App Selector Slicer
  • 3 KPI Cards (Instance count, CPU avg, RAM avg)
  • Instance Bar Chart (Top 15 by CPU+RAM)
  • Instance Details Table (with percentiles & recommendations)

⚙️ Page 3: Instances & Config (Full Configuration)

  • App Filter Dropdown
  • Flavor Filter Dropdown
  • CPU Distribution Chart (by Flavor)
  • RAM Distribution Chart (by Flavor)
  • Complete Instance Configuration Table

🗂️ Data Model

5 Tables with Relationships

Flavors (Lookup)
  ├─ FlavorID, FlavorName, CPU_vCPU, RAM_GB, Cost_Monthly
  │
  └─→ Instances (Dimension)
        ├─ InstanceID, AppID, FlavorName, Status, Corrected
        ├─ RecFlavorName, CPU_vCPU, RAM_GB, CreatedDate
        │
        ├─→ Metrics (Fact - Large)
        │     ├─ MetricID, InstanceID, MeasureDate
        │     ├─ CPU_P50/P90/P95/P99, RAM_P50/P90/P95/P99
        │     └─ (1500+ daily records)
        │
        └─→ Apps (Lookup)
              ├─ AppID, AppName, Domain, Environment
              │
              └─→ Timeline (Time)
                    ├─ TimelineID, Month, SnapshotDate
                    ├─ TotalInstances, CriticalCount
                    ├─ OverConsumptionCount, OKCount, etc.
                    └─ CorrectionPct, HealthScore
